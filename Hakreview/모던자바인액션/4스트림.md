# 스트림?
현업에서 스트림을 정말 많이 사용한다. Collection 변환시, Entity를 다른 객체와 조합하여 DTO 생성시, Map 만들떄 등등... 어쩔때는 N중첩 스트립을 풀어야 할때도 있다....
앞으로 좀 더 스트림을 잘 다루기 위해 정리한다.
스트림을 이용하면 선언형으로 컬렉션 데이처를 처리할 수 있으며 멀티쓰레드 코드 구현없이 데이터를 병렬처리할 수 있다. 

** java8이전 버전의 코드 
```java
List<Dish> dishes = new ArrayList<>();
for(Dish d : menu){
    if(d.getCalories() <400){
        dishes.add(d);
    }
}
```

** java8이후 버전 
```java
List<Dish> dishes = menu.stream()
        .filter(d -> d.getcalories() <400)
        .collect(toList());

```
내가 보통 사용하는 스트림은 이정도 수준의 스트림이었다. 하지만 조금 더 유용하게 다르기위헤 스트림에 대해 `모던자바인액션` 을 정리하려한다.
